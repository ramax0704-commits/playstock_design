# 조각 구매

> 사용자가 원하는 수량의 조각을 선택하고 구매를 진행하는 기능입니다. 수량 선택, 가격 계산, 결제 처리를 포함합니다.

**관련 UI 페이지**: 
- SCR-I003 거래 패널 (수량 선택)
- SCR-I005 구매 완료 (결제 결과)

---

## 데이터 모델

### Purchase (구매 주문)

| 필드 | 타입 | 필수 여부 | 설명 |
|---|---|---|---|
| `id` | UUID | 필수 | 구매 주문 ID |
| `userId` | UUID | 필수 | 사용자 ID (외래키) |
| `channelId` | UUID | 필수 | 채널 ID (외래키) |
| `quantity` | number | 필수 | 구매 수량 |
| `unitPrice` | number | 필수 | 단가 (구매 시점의 가격) |
| `totalPrice` | number | 필수 | 총 가격 = quantity × unitPrice |
| `status` | PurchaseStatus | 필수 | 주문 상태 |
| `paymentMethod` | string | 필수 | 결제 수단 (예: "card", "account_transfer") |
| `transactionId` | string | 필수 | 결제 거래 ID |
| `createdAt` | Date | 필수 | 주문 생성 날짜 |
| `completedAt` | Date | 선택 | 완료 날짜 (status = "COMPLETED"일 때) |

**열거형(Enum) 값**:
```
PurchaseStatus: "PENDING" | "COMPLETED" | "FAILED" | "CANCELLED"
```

### PurchaseCart (장바구니 항목)

| 필드 | 타입 | 필수 여부 | 설명 |
|---|---|---|---|
| `channelId` | UUID | 필수 | 채널 ID |
| `quantity` | number | 필수 | 원하는 수량 |
| `unitPrice` | number | 필수 | 현재 단가 |

## API 엔드포인트

### 구매 정보 조회 (거래 패널 초기 로드)
```
GET /api/purchases/preview/:channelId
```

**인증 필요**: 예

**경로 파라미터**:

| 파라미터 | 타입 | 필수 | 설명 |
|---|---|---|---|
| `channelId` | UUID | 필수 | 채널 ID |

**응답** (성공 - 200):
```json
{
  "data": {
    "channelId": "uuid",
    "channelName": "뜬뜬 DdeunDdeun",
    "channelThumbnail": "https://...",
    "unitPrice": 2500,
    "availableQuantity": 5000,
    "saleProgress": 66,
    "minQuantity": 1,
    "maxQuantity": 100,
    "quickSelectOptions": [1, 5, 10, 20]
  }
}
```

### 구매 수량 선택 및 가격 계산
```
POST /api/purchases/calculate
```

**인증 필요**: 예

**요청 본문**:
```json
{
  "channelId": "uuid",
  "quantity": 10
}
```

**응답** (성공 - 200):
```json
{
  "data": {
    "channelId": "uuid",
    "quantity": 10,
    "unitPrice": 2500,
    "subtotal": 25000,
    "discount": 0,
    "fee": 0,
    "tax": 0,
    "totalPrice": 25000,
    "message": "25,000원 결제하기"
  }
}
```

### 구매 요청 (결제 전)
```
POST /api/purchases
```

**인증 필요**: 예

**요청 본문**:
```json
{
  "channelId": "uuid",
  "quantity": 10,
  "totalPrice": 25000,
  "paymentMethod": "card"
}
```

**응답** (성공 - 201):
```json
{
  "data": {
    "id": "purchase_uuid",
    "status": "PENDING",
    "redirectUrl": "https://payment-gateway.example.com/...",
    "transactionId": "tx_12345"
  }
}
```

### 구매 완료 (결제 후 콜백)
```
POST /api/purchases/:id/complete
```

**인증 필요**: 예

**경로 파라미터**:

| 파라미터 | 타입 | 필수 | 설명 |
|---|---|---|---|
| `id` | UUID | 필수 | 구매 주문 ID |

**요청 본문**:
```json
{
  "transactionId": "tx_12345",
  "paymentConfirmation": "confirmation_code"
}
```

**응답** (성공 - 200):
```json
{
  "data": {
    "id": "purchase_uuid",
    "status": "COMPLETED",
    "channelId": "uuid",
    "channelName": "뜬뜬 DdeunDdeun",
    "quantity": 10,
    "totalPrice": 25000,
    "completedAt": "2026-05-19T12:34:56Z",
    "nextSteps": [
      {
        "label": "채널 상세 보기",
        "route": "/channels/uuid"
      },
      {
        "label": "마이페이지",
        "route": "/mypage"
      }
    ]
  }
}
```

**에러 응답**:

| 코드 | 상황 |
|---|---|
| 400 | 잘못된 수량 또는 가격 |
| 401 | 미인증 |
| 404 | 채널 또는 구매 주문 없음 |
| 409 | 판매 수량 부족 |
| 422 | 결제 검증 실패 |

## 비즈니스 로직

### 수량 선택 & 가격 계산
- 사용자가 Stepper 또는 Quick Select로 수량 선택
- 실시간으로 `POST /api/purchases/calculate` 호출하여 가격 계산
- UI에서 동적으로 가격 업데이트

### Quick Select 옵션
- 미리 정의된 수량: [1, 5, 10, 20] (디자인에서 확인)
- 각 선택지는 빠른 구매를 유도

### 최소/최대 수량 제한
- 최소 1개, 최대 100개 (또는 판매 가능한 전체 수량)

### 결제 플로우
1. 사용자가 "결제하기" 클릭
2. `POST /api/purchases` 호출 → 구매 주문 생성 (status: PENDING)
3. 결제 게이트웨이로 리다이렉트
4. 결제 완료 후 결제사에서 콜백
5. `POST /api/purchases/:id/complete` 호출 → 주문 상태 변경 (status: COMPLETED)
6. SCR-I005 구매 완료 페이지로 이동

## 유효성 검사 규칙

| 필드 | 규칙 | 에러 메시지 |
|---|---|---|
| `channelId` | UUID 형식 | "유효하지 않은 채널 ID입니다" |
| `quantity` | 1 이상의 정수 | "수량은 1 이상이어야 합니다" |
| `quantity` | 최대값 이하 | "판매 수량을 초과했습니다" |
| `totalPrice` | quantity × unitPrice와 일치 | "가격이 일치하지 않습니다" |
| `paymentMethod` | "card" \| "account_transfer" | "유효하지 않은 결제 수단입니다" |
