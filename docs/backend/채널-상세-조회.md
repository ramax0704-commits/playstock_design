# 채널 상세 조회

> 사용자가 특정 채널의 상세 정보를 탭별(조각정보, 가치평가, 수익 구조)으로 조회할 수 있는 기능입니다.

**관련 UI 페이지**: SCR-I002 채널 상세 (조각정보, 가치평가, 수익 구조)

---

## 데이터 모델

### Channel (채널)

| 필드 | 타입 | 필수 여부 | 설명 |
|---|---|---|---|
| `id` | UUID | 필수 | 채널 고유 식별자 |
| `name` | string | 필수 | 채널명 (예: "뜬뜬 DdeunDdeun") |
| `thumbnailUrl` | string | 필수 | 채널 썸네일 이미지 URL |
| `description` | string | 선택 | 채널 설명 |
| `isLiked` | boolean | 필수 | 찜 여부 (사용자별) |
| `likesCount` | number | 필수 | 총 찜 개수 |
| `createdAt` | Date | 필수 | 채널 생성 날짜 |

### ChannelDetail (채널 상세 정보)

#### 조각정보 탭
| 필드 | 타입 | 필수 여부 | 설명 |
|---|---|---|---|
| `fragmentPrice` | number | 필수 | 조각 단가 |
| `fragmentCount` | number | 필수 | 보유 조각 수 |
| `totalFragments` | number | 필수 | 전체 조각 수 |
| `saleProgress` | number | 필수 | 판매 진행률 (0-100) |
| `saleEndDate` | Date | 필수 | 판매 종료 예정일 |
| `grassInfo` | GrassInfo[] | 선택 | 조각 잔디 정보 |

#### 가치평가 탭
| 필드 | 타입 | 필수 여부 | 설명 |
|---|---|---|---|
| `aiValuation` | AIValuation | 필수 | AI 가치평가 결과 |
| `categoryComparison` | CategoryComparison | 필수 | 카테고리별 비교 데이터 |

#### 수익 구조 탭
| 필드 | 타입 | 필수 여부 | 설명 |
|---|---|---|---|
| `monthlyRevenue` | number | 필수 | 예상 월 수익 |
| `revenueBreakdown` | RevenueBreakdown[] | 필수 | 수익 항목별 상세 정보 |
| `investmentType` | InvestmentType | 필수 | 투자 성향 (안정 배당형 / 공격 성장형) |

### AIValuation

| 필드 | 타입 | 필수 여부 | 설명 |
|---|---|---|---|
| `score` | number (0-100) | 필수 | 평가 점수 |
| `gradeLabel` | string | 필수 | 등급 표시 (A, B, C 등) |
| `description` | string | 필수 | 평가 설명 |

### RevenueBreakdown

| 필드 | 타입 | 필수 여부 | 설명 |
|---|---|---|---|
| `category` | string | 필수 | 수익 항목명 (예: "광고 수익") |
| `amount` | number | 필수 | 예상 금액 |
| `percentage` | number | 필수 | 비율 (%) |

**열거형(Enum) 값**:
```
InvestmentType: "STABLE_DIVIDEND" | "AGGRESSIVE_GROWTH"
```

## API 엔드포인트

### 채널 상세 정보 조회
```
GET /api/channels/:id
```

**인증 필요**: 예

**경로 파라미터**:

| 파라미터 | 타입 | 필수 | 설명 |
|---|---|---|---|
| `id` | UUID | 필수 | 채널 ID |

**쿼리 파라미터**:

| 파라미터 | 타입 | 필수 | 설명 |
|---|---|---|---|
| `tab` | string | 아니오 | 탭 종류 ("fragment" / "valuation" / "revenue", 기본값: "fragment") |

**응답** (성공 - 200):
```json
{
  "data": {
    "id": "uuid",
    "name": "뜬뜬 DdeunDdeun",
    "thumbnailUrl": "https://...",
    "description": "채널 설명",
    "isLiked": false,
    "likesCount": 1234,
    "detail": {
      "fragmentPrice": 2500,
      "fragmentCount": 100,
      "totalFragments": 10000,
      "saleProgress": 66,
      "saleEndDate": "2026-06-30T00:00:00Z",
      "grassInfo": [
        {
          "title": "조각 잔디",
          "imageUrl": "https://..."
        }
      ],
      "aiValuation": {
        "score": 85,
        "gradeLabel": "A",
        "description": "높은 성장 잠재력"
      },
      "categoryComparison": {
        "categories": ["카테고리1", "카테고리2"],
        "values": [85, 78]
      },
      "monthlyRevenue": 1980,
      "revenueBreakdown": [
        {
          "category": "광고 수익",
          "amount": 1200,
          "percentage": 60
        },
        {
          "category": "후원 수익",
          "amount": 600,
          "percentage": 30
        }
      ],
      "investmentType": "STABLE_DIVIDEND"
    },
    "createdAt": "2026-01-01T00:00:00Z"
  }
}
```

**에러 응답**:

| 코드 | 상황 |
|---|---|
| 400 | 잘못된 채널 ID 형식 |
| 401 | 미인증 |
| 404 | 채널을 찾을 수 없음 |

### 채널 찜하기 토글
```
POST /api/channels/:id/like
```

**인증 필요**: 예

**경로 파라미터**:

| 파라미터 | 타입 | 필수 | 설명 |
|---|---|---|---|
| `id` | UUID | 필수 | 채널 ID |

**요청 본문**: 없음

**응답** (성공 - 200):
```json
{
  "data": {
    "isLiked": true,
    "likesCount": 1235
  }
}
```

## 비즈니스 로직

### 탭별 데이터 조회
- **조각정보 탭**: 판매 가격, 판매 진행률, 판매 종료일 등 거래 관련 정보
- **가치평가 탭**: AI 알고리즘 기반 채널 평가 점수 및 카테고리 비교
- **수익 구조 탭**: 예상 월 수익, 수익 항목별 분석, 투자 성향

모든 탭 데이터는 한 번의 API 호출로 전달되며, 클라이언트에서 탭 선택에 따라 표시

### 찜하기
- 사용자가 하트 아이콘 클릭 시 찜 상태 토글
- UI에서 즉시 반영 (낙관적 업데이트)
- 백엔드에서 비동기로 저장

### 수익 구조 계산
- 월 수익 = 예상 수익 항목 합계
- 각 항목의 백분율은 합계가 100%가 되도록 정규화
- 수정 데이터: "본 자료는 공모가 기준으로 추정된 데이터입니다"

## 유효성 검사 규칙

| 필드 | 규칙 | 에러 메시지 |
|---|---|---|
| `id` | UUID 형식 | "유효하지 않은 채널 ID입니다" |
| `tab` | "fragment" \| "valuation" \| "revenue" | "유효하지 않은 탭입니다" |
| `fragmentPrice` | 0 초과 | "가격은 0보다 커야 합니다" |
| `saleProgress` | 0-100 사이의 정수 | "진행률은 0~100 사이여야 합니다" |
