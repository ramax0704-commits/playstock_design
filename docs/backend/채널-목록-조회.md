# 채널 목록 조회

> 사용자가 현재 거래 중인 채널과 곧 오픈될 예정인 채널을 탭으로 필터링하여 조회할 수 있는 기능입니다. 상단 캐러셀에는 인기 채널이 표시됩니다.

**관련 UI 페이지**: SCR-I001 채널 리스트 (거래 채널 탭, 오픈 예정 탭)

---

## 데이터 모델

### Channel (채널)

| 필드 | 타입 | 필수 여부 | 설명 |
|---|---|---|---|
| `id` | UUID | 필수 | 채널 고유 식별자 |
| `name` | string | 필수 | 채널 이름 |
| `status` | ChannelStatus | 필수 | 채널 상태 (거래 중 / 오픈 예정) |
| `thumbnailUrl` | string | 필수 | 채널 썸네일 이미지 URL |
| `isPopular` | boolean | 필수 | 인기 채널 여부 (캐러셀 표시 기준) |
| `description` | string | 선택 | 채널 설명 |
| `createdAt` | Date | 필수 | 채널 생성 날짜 |
| `updatedAt` | Date | 필수 | 채널 마지막 수정 날짜 |

**열거형(Enum) 값**:
```
ChannelStatus: "TRADING" | "UPCOMING"
```

- `Channel`은 여러 개의 `ChannelCard` (리스트 아이템)를 가짐

### ChannelCard (채널 카드 아이템)

| 필드 | 타입 | 필수 여부 | 설명 |
|---|---|---|---|
| `channelId` | UUID | 필수 | 채널 ID (외래키) |
| `title` | string | 필수 | 카드에 표시될 제목 |
| `image` | string | 필수 | 카드 썸네일 URL |
| `badge` | string | 선택 | 배지 텍스트 (예: "소진임박", "인기") |


## API 엔드포인트

### 채널 목록 조회 (필터링)
```
GET /api/channels
```

**인증 필요**: 예

**쿼리 파라미터**:

| 파라미터 | 타입 | 필수 | 설명 |
|---|---|---|---|
| `status` | ChannelStatus | 아니오 | 채널 상태 필터 ("TRADING" 또는 "UPCOMING") |
| `page` | number | 아니오 | 페이지 번호 (기본값: 1) |
| `limit` | number | 아니오 | 페이지당 항목 수 (기본값: 10) |
| `popular` | boolean | 아니오 | 인기 채널만 필터 (캐러셀용, true 시 최대 5개 반환) |

**응답** (성공 - 200):
```json
{
  "data": [
    {
      "id": "uuid",
      "name": "channel-name",
      "status": "TRADING",
      "thumbnailUrl": "https://...",
      "isPopular": true,
      "description": "채널 설명",
      "cards": [
        {
          "channelId": "uuid",
          "title": "제목",
          "image": "https://...",
          "badge": "소진임박",
          "badgeColor": "card-pink"
        }
      ],
      "createdAt": "2026-05-19T00:00:00Z",
      "updatedAt": "2026-05-19T00:00:00Z"
    }
  ],
  "meta": {
    "total": 15,
    "page": 1,
    "limit": 10,
    "pages": 2
  }
}
```

**에러 응답**:

| 코드 | 상황 |
|---|---|
| 400 | 잘못된 쿼리 파라미터 (예: 잘못된 status 값) |
| 401 | 미인증 (로그인 필요) |
| 403 | 권한 없음 |
| 500 | 서버 에러 |

### 인기 채널 조회 (캐러셀용)
```
GET /api/channels?popular=true
```

**인증 필요**: 예

**쿼리 파라미터**:

| 파라미터 | 타입 | 필수 | 설명 |
|---|---|---|---|
| `popular` | boolean | 필수 | true로 설정 |

**응답** (성공 - 200):
```json
{
  "data": [
    {
      "id": "uuid",
      "name": "인기채널1",
      "status": "TRADING",
      "thumbnailUrl": "https://...",
      "isPopular": true
    },
    {
      "id": "uuid",
      "name": "인기채널2",
      "status": "TRADING",
      "thumbnailUrl": "https://..."
    }
  ],
  "meta": {
    "total": 2,
    "limit": 5
  }
}
```

## 비즈니스 로직

### 탭 필터링
사용자가 탭을 전환하면 다음 로직이 작동합니다:
- **거래 채널 탭 선택**: `status = "TRADING"` 필터 적용, 3개 리스트 아이템 표시
- **오픈 예정 탭 선택**: `status = "UPCOMING"` 필터 적용, 2개 리스트 아이템 표시

UI의 탭 상태 변경은 클라이언트 측에서 처리되며, API는 `status` 파라미터에 따라 데이터를 필터링합니다.

### 페이지네이션
- 기본적으로 페이지당 10개 항목 표시
- 사용자가 "더보기" 버튼 클릭 시: 다음 페이지로 이동 (또는 무한 스크롤)

### 인기 채널
- `isPopular = true`인 채널만 캐러셀에 표시
- 최대 5개까지만 반환 (캐러셀 UI 크기 고려)
- 거래 채널과 오픈 예정 상관없이 모두 표시 가능

## 유효성 검사 규칙

| 필드 | 규칙 | 에러 메시지 |
|---|---|---|
| `status` | "TRADING" 또는 "UPCOMING"만 허용 | "유효하지 않은 상태입니다" |
| `page` | 1 이상의 정수 | "페이지는 1 이상이어야 합니다" |
| `limit` | 1 이상 100 이하 | "페이지당 항목 수는 1~100 사이여야 합니다" |
| `popular` | boolean | "popular은 true/false만 허용됩니다" |
