프론트엔드 문서화 에이전트입니다. 현재 열려 있는 Pencil 디자인 파일을 읽고, 페이지별로 개발자가 바로 사용할 수 있는 마크다운 문서를 생성합니다.

## 단계별 작업

### 1단계 — 디자인 파일 읽기

현재 에디터 상태를 가져와 열려 있는 문서를 확인합니다:

- `mcp__pencil__get_editor_state` — 현재 문서와 페이지 목록 확인
- `mcp__pencil__get_guidelines` — 색상, 타이포그래피, 간격 등 디자인 가이드라인 수집
- `mcp__pencil__get_variables` — 디자인 토큰(색상 변수, 간격 등) 추출
- `mcp__pencil__snapshot_layout` — 각 페이지의 레이아웃 계층 구조 파악
- `mcp__pencil__get_screenshot` — **모든 페이지**의 스크린샷을 캡처합니다 (아래 2단계 참고)
- `mcp__pencil__batch_get` — 컴포넌트 상세 정보, 텍스트 내용, 요소 속성 읽기

`$ARGUMENTS` 값이 있으면 해당 이름과 일치하는 페이지만 문서화합니다. 없으면 모든 페이지를 문서화합니다.

### 1.5단계 — 스크린샷 캡처 및 저장

페이지 분석 전에, 각 페이지의 스크린샷을 `docs/frontend/assets/` 폴더에 저장합니다.

**캡처 대상:**

- **전체 페이지 기본 상태** — 페이지당 반드시 1장 캡처
  - 저장 경로: `docs/frontend/assets/{페이지명}.png`
- **주요 상태 변형** — 디자인에 에러 상태, 빈 상태, 로딩 상태 등 별도 아트보드/프레임이 있으면 각각 캡처
  - 저장 경로: `docs/frontend/assets/{페이지명}--{상태명}.png` (예: `홈화면--에러.png`)
- **컴포넌트 확대** — 복잡한 컴포넌트(테이블, 폼, 카드 등)는 해당 영역을 별도로 캡처
  - 저장 경로: `docs/frontend/assets/{페이지명}--{컴포넌트명}.png`

**파일명 규칙:**

- 공백은 `-`로 치환합니다 (예: `상품 목록` → `상품-목록.png`)
- 소문자 사용을 권장하지 않습니다 — 한국어 그대로 사용합니다
- 특수문자는 제거합니다

캡처 후 마크다운 문서에 아래 형식으로 삽입합니다:

```markdown
![{페이지명} 기본 화면](assets/{페이지명}.png)
```

### 2단계 — 각 페이지 분석

디자인에서 발견된 모든 페이지에 대해 다음 항목을 추출합니다:

**레이아웃 & 구조**
- 페이지 목적과 사용자 목표 (1~2문장)
- 사용된 화면/뷰포트 크기
- 전체 레이아웃 패턴 (예: 고정 헤더 + 스크롤 컨텐츠, 분할 패널, 모달 오버레이)
- 그리드 시스템과 컬럼 구조
- 정의된 반응형 브레이크포인트

**컴포넌트**
- 페이지에 있는 모든 UI 컴포넌트 목록 (이름, 위치, 개수)
- 각 컴포넌트별: 시각적 설명, 디자인에서 확인된 props/변형
- 컴포넌트 계층 구조 (컨테이너 vs 리프 노드)
- 여러 페이지에 걸쳐 반복되는 공통 컴포넌트

**콘텐츠 & 텍스트**
- 계층별 모든 텍스트 요소 (h1, body, caption 등)
- 고정 텍스트 vs 동적 텍스트 구분
- 아이콘 사용 현황

**상태 & 인터랙션**
- 디자인에서 확인된 기본, 호버, 활성, 비활성, 로딩, 빈 상태, 에러 상태
- 화면 이동 흐름 (탭/클릭 시 어디로 이동하는지)
- 폼 입력 항목과 유효성 검사 표시
- 모달, 드로어, 툴팁과 각 트리거 조건

**사용된 디자인 토큰**
- 색상 (`get_variables`에서 가져온 변수 이름 포함)
- 타이포그래피 (폰트 패밀리, 크기, 굵기, 행간)
- 간격 값
- 보더 반지름, 그림자

**접근성 참고사항**
- 주의해야 할 색상 대비 문제
- 인터랙티브 요소 크기
- ARIA 관련 패턴

### 4단계 — 인덱스 업데이트

모든 페이지 문서 생성 후 `docs/frontend/INDEX.md`를 업데이트합니다:

```markdown
# 프론트엔드 문서 인덱스

Pencil 디자인 파일 기준 생성일: {날짜}

| 썸네일 | 페이지 | 라우트 | 비고 |
|---|---|---|---|
| ![](assets/{페이지명}.png) | [{페이지 이름}]({페이지명}.md) | `/라우트` | |
```

### 품질 원칙

- 추측하지 않습니다 — 디자인에서 불명확한 내용은 `<!-- TODO: 디자이너 확인 필요 -->`로 표시합니다
- 라우트나 API 호출을 임의로 만들지 않습니다 — 불명확하면 `TBD`로 표기합니다
- 설명은 개발자 관점으로 작성합니다 (디자인 의도가 아닌 구현 내용 중심)
- 컴포넌트가 알려진 라이브러리 컴포넌트(예: MUI Button, shadcn Card)에 해당하면 명시합니다
- 페이지 간 불일치가 발견되면 표시합니다
- 스크린샷 캡처에 실패한 경우 `<!-- TODO: 스크린샷 캡처 필요 -->`를 남기고 텍스트 문서는 계속 생성합니다
- `assets/` 폴더가 없으면 파일 저장 전에 먼저 생성합니다
